# Рыночное окружение RiskCalc — анализ источников и план загрузки

**Дата:** 2026-06-11
**Метод:** инвентаризация потребностей всех моделей реестра × живые пробы ISS MOEX
(эндпоинты опрошены 2026-06-11, числа ниже — фактические ответы API).

---

### 1.2 Волатильность

| Данные | Потребители | Источник | Статус |
|---|---|---|---|
| Equity/Index IV | ваниль, экзотика, autocall | FORTS опционы: **VOLATILITY пуст в EOD-выдаче** (причина «0 точек» в инжесте), но `SETTLEPRICE` есть по 38 162 инструментам; страйк в SECID, OPTIONTYPE/LASTTRADEDATE в метаданных → **имплаим сами** (наш `implied_vol` + Black76 на фьючерс), затем SVI-фит | ⚙️ переделать инжест |
| FX smile (RR/BF) | fx_option_smile, Malz | Si-опционы: **1170 инструментов** → полный смайл → RR/BF из имплая | ⚙️ то же |
| RVI (vol-индекс) | риск-факторы | ISS index history | ✅ грузится |
| Свопционный куб (IRVOL) | бермудцы, CMS, капы | **Бесплатного источника НЕТ** (OTC, без экрана). Решение: куб остаётся expert-input; прокси-валидация через realized vol КБД-истории | 🔶 ручной ввод |

| Данные | Потребители | Источник | Статус |
|---|---|---|---|

| CDS-спреды | — | Рынка нет с 2022 | 🔶 N/A |

### 1.5 Честный итог по покрытию
Бесплатно закрывается **всё, кроме**: свопционного куба (expert-input + realized-прокси),
CDS (рынка нет — hazard из корп-спредов), форвардных дивидендов (TTM-прокси),
recovery (допущение). Все четыре уже имеют ручные/derived пути в коде — ни одна
модель не остаётся без входов.

## 2. План реализации

### Этап I — Загрузка (≈1 неделя)
1. **Бэкфил истории** (день): цикл `run_eod_ingest` по торговым дням (КБД по `date=`,
   индексы IMOEX/RGBI/RVI/RUSFAR\*, history топ-50 тикеров TQBR) → наполнение
   `time_series` глубиной ≥ 250 дней для VaR. CLI-флаги `--equities`, `--backfill N`.
2. **TQCB-инжест** (день): котировки+YTM+дюрация 2983 корп-бондов; бакетирование
   кривых YTM-кластерами по дюрации (рейтингов в ISS нет — кластеризация
   по уровню спреда; рейтинги АКРА/ЭкспертРА — отдельная фаза, скрейпинг).
3. **Реальная кривая + ИПЦ** (день): реальные YTM четырёх ОФЗ-ИН → real curve;
   парсер ИПЦ Росстат/ЦБ → базовая индексация; breakeven = КБД − real.
4. **FX-блок** (полдня): стрипы Si/CR → форвардная кривая; CBR official rates →
   USD/EUR фиксы.
5. **Опционные поверхности** (1–2 дня): новый `ingest_option_vol_surface`:
   settle-цены + метаданные → имплай IV нашим движком (Black76 на фьючерсный
   settle) → фильтры качества (OI>0, не глубокие OTM) → SVI-фит → поверхности
   Si (FX) и RTS/индекс (equity); RR/BF из смайла Si.
6. **Bondization** (полдня): купоны/амортизация/оферты для точного прайсинга
   реальных выпусков.

### Этап II — Автоматизация (2–3 дня)
1. launchd-джоб (mac) ежедневного EOD-инжеста ~19:30 МСК + бэкфил пропусков.
2. Отчёт качества в `ingest_log`: полнота по источникам, свежесть, алерты в лог.
3. GitHub Actions CI: pytest на каждый push (на synthetic data) + nightly
   smoke-инжест с проверкой структуры ответов ISS (ловит изменения API).
4. Приложение: автоподключение `data/market_data.sqlite` при старте —
   реальный снапшот по умолчанию, demo как фолбэк.

### Этап III — Визуализация (3–5 дней)
1. Market Data workspace: реальная КБД с историей (анимация по датам),
   корп-спреды по бакетам, real curve + breakeven-панель.
2. Vol Explorer: имплаенные поверхности (смайлы по экспирациям, SVI-фит поверх
   сырых точек, ATM-терм-структура).
3. Data Health dashboard: свежесть/полнота/качество по каждому источнику,
   история инжестов, пропуски календаря.
4. Time Series explorer: ряды факторов VaR с доходностями и корреляциями.

### Зависимости и порядок
I.1 → II.1 (бэкфил до автоматизации); I.2 → hazard-кривые из реальных спредов;
I.3 → инфляционный стек на реальных данных; I.5 → калибровки Heston/SABR/LV
на реальных смайлах. Этап III после I (визуализировать есть что), параллельно II.
