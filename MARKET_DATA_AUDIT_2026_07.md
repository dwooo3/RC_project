## 1. Дефекты кода (подтверждены)

| ID | Серьёзность | Что | Где |
|----|------|-----|-----|
| A1 | P0 | **Stale-write race в `select()`**: два быстрых клика по списку → два конкурентных `select`; поздний клик может финишировать раньше, ранний перезаписывает `entity`/`bars` устаревшей бумагой (пара «заголовок одной, график другой»). Нет guard после `await`. | `MarketEntityView.swift` `MarketEntityVM.select/changeRange/changeInterval` |
| A2 | P0 | **Индексы: акция MOEX выдаётся как индекс.** `_list_indices` аллоулистит базу `MOEX`, но `MOEX:price` — это бэкфилл АКЦИИ Московской биржи. При этом настоящие индексы из БД не показаны: `RVI` (1256 т.), `RUCBTRNS` (1249), `RUSFAR` (1226) — есть в `time_series`, отсутствуют в UI. | `api/market_entity.py::_list_indices` |
| A3 | P1 | **Мёртвый LIVE для FX.** FX-инструменты — CBR-фиксинги (`USDRUB/EURRUB/CNYRUB`, board=cbr), а intraday-роут шлёт их в `currency/selt`, где таких secid нет → пустой график с горящим «LIVE». Нужен гейтинг интервалов per-category или маппинг `USDRUB→USD000UTSTOM`. | `api/intraday.py` + `MarketEntityView.rangeBar` |
| A4 | P1 | **VM пересоздаётся на каждый свитч под-вкладки**: `.id(instrument)` создаёт новый `MarketEntityVM` → рефетч всего списка (bonds ≈ 700KB, 3021 строка) при каждом возврате на вкладку. Плюс `onChange(of: category)` в `MarketEntityView` — мёртвый код (`category` — `let`, никогда не меняется у живого VM). | `MarketScreen.swift:208`, `MarketEntityView.swift:149` |
| A5 | P1 | **Сброс зума при toggle SMA/Log**: смена этих флагов идёт через полный `render()` + `fitContent()` → выставленный зум/пан теряется. TradingView сохраняет вьюпорт. | `TradingChart.swift` (JS `render`) |
| A6 | P2 | **Yield-режим + интрадей = пустой график** (у ISS-свечей нет yield). Кнопку Yield надо прятать/дизейблить при интрадей-интервале. | `TradingChart.swift` controls |
| A7 | P2 | Пустой график вне торгов/при недоступном ISS — просто чёрная область, без сообщения «нет интрадей-данных / торги закрыты». | `TradingChart.swift` (JS empty-state) |
| A8 | P2 | Объём-панель для рядов без объёма (индексы) рисует плоский ноль — прятать histogram, когда все volume null/0. | `TradingChart.swift` (JS) |
| A9 | P2 | `rawdata.table_rows` — `LIMIT` без `ORDER BY`: страница недетерминирована; для `ingest_log` полезнее хвост (`ORDER BY rowid DESC`). | `api/rawdata.py` / `market_data_db.table_rows` |
| A10 | P2 | `intraday._ENGINE_MARKET` не покрывает `commodities` явно (работает через `forts`, но маппинг категории делается на Swift-стороне — хрупко при новом потребителе API). Задокументировать/принимать категорию. | `api/intraday.py` |

## 2. Дизайн-косяки

| ID | Что |
|----|-----|
| D1 | Шапка вкладки не отвечает рекомендациям (§4 doc): нет глобального поиска инструмента, нет `source`, нет «обновлено HH:MM» (есть только мелкое «данные на YYYY-MM-DD»). `fetch_ts` снапшота в БД есть — не выводится. |
| D2 | Список облигаций показывает только price+Δ%: **YTM и accrued есть в `bond_quotes` на 100% строк** и не показаны (см. B1). Строка списка беднее, чем позволяет БД. |
| D3 | Локализация смешана: «Bonds/Equities/Futures…», «Candles/Line/Yield» (EN) рядом с «Качество», «Валюта: все», «Карточка» (RU). Нужен единый словарь (RU-first). |
| D4 | Overview без «недавно просмотренных» и без ключевых индексов (IMOEX/RGBI/RVI/Brent) — только FX. Плитки не показывают динамику (Δ%). |
| D5 | Vol surface: список базовых активов без поиска/сортировки; таблица опционов длинная без сворачивания по экспирациям. |
| D6 | Экспорт CSV есть только в Instruments; кривые, vol-таблица, OTC-тенора — не выгружаются. |

## 4. Неиспользуемая мощь данных (главное)

Данные УЖЕ в БД, движки УЖЕ валидированы — не хватает только выдачи:

| ID | Данные | Что можно показать |
|----|--------|--------------------|
| B1 | `bond_quotes.ytm/accruedint/wap` (3071/3071/2752 строк) | Колонки **YTM** в списке облигаций + Accrued/WAP в детали; сортировка списка по YTM. |
| B2 | `GCURVE`/`ZCB_OFZ_RUB` + YTM бумаги | **G-spread** per bond (YTM − zero(T_mat)) — колонка + в деталь. Кривая уже в снапшоте, интерполяция тривиальна. |
| B3 | `instrument_ref` futures chain (last + last_trade_date, 6+ контрактов на asset) | **Фьючерсная кривая** (term structure) в карточке фьючерса — line chart settle vs expiry. Данные готовы. |
| B4 | `time_series`: RVI, RUCBTRNS, RUSFAR, RGBI | Полноценный Indices-таб (сейчас 3 из 7 реальных рядов, один из которых — акция). RVI — индекс волатильности, просится на Overview. |
| B5 | `dividends` (181 эмитент) | **Дивидендная доходность** в списке/детали акций (sum(12m)/price). |
| B6 | `price_history` (1.8M строк) | Пер-инструментная аналитика: 52w high/low, реализованная вола 30д, max drawdown — строка статистики в детали. |
| B7 | B6 + vol surface | **RV vs IV**: реализованная вола базового актива против ATM implied (для 19 underlying'ов) — уникальная фича-«эдж», всё уже посчитано. |
| B8 | SABR per expiry (все 19 активов) | **25Δ RR/BF skew-аналитика для всех** активов (не только FX): OTC-механика `rr_bf_25delta` работает на любом смайле → таблица скewа в Volatility. |
| B9 | ISS index candles (проверено: IMOEX/RGBI 60м отдаются) | Интрадей для Indices-таба — включить маппинг (уже есть) + снять гейтинг. |

## 5. Сводный план реализации

Порядок выбран так: сначала снять ложь в UI (баги), затем дешёвые данные-фичи с
максимальным эффектом, затем UX-слой.

### Фаза 1 — корректность (PLAN_A, ~1 заход)
A1 select-guard → A2 честные индексы (+RVI/RUCBTRNS/RUSFAR, минус MOEX-акция)
→ A3 intraday-гейтинг per-category (+индексы B9) → A4 кэш VM → A5 сохранение
зума → A6–A8 полировка чартов. Критерий: быстрые клики по списку не дают
рассинхрона; Indices показывает 6 настоящих индексов с интрадеем; LIVE не горит
там, где данных нет; зум переживает toggles.

### Фаза 2 — мощь данных (PLAN_B, 2–3 захода)
B1+B2 (bonds: YTM/G-spread + сортировка) → B3 (futures curve) → B5 (див-доходность)
→ B6+B7 (стат-строка + RV vs IV) → B8 (skew для всех активов). Каждый пункт —
отдельный commit со скрин-проверкой.

### Фаза 3 — UX (PLAN_C, 1–2 захода)
C1 глобальный поиск + шапка (source/updated) → C2 Overview 2.0 (индексы, Δ%,
недавние) → C3 экспорт кривых/vol → C4 локализация → C5 compare-overlay
(опционально).

### Не делаем (осознанно)
- Снапшот-селектор в шапке — конфликт со snapshot-less дизайном.
- Тик-стриминг/вебсокеты — ISS-poll 15с достаточен уровню данных.
- Пер-инструментный ingest OI-истории — данных в ISS EOD нет.
