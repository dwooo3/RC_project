# Аудит данных и план инфраструктуры визуализации

**Дата:** 2026-06-13
**Метод:** инвентаризация локальной БД × сверка с UI × живые пробы ISS × потребности роадмапа.
**Контекст:** рыночное окружение построено (этапы I–III), данные грузятся и частично
визуализируются. Этот документ — что собрано vs показано, что ещё доступно бесплатно,
что понадобится будущей разработке, и как построить графический слой для ВСЕХ данных.

### 1.2 Главные разрывы «собрано → не показано»
1. **Облигации (6155 котировок + 5500 cashflow-строк) нигде не видны** — самый большой
   неиспользуемый массив. Нет bond-explorer: ни YTM/дюрация скаттера, ни графика купонов.
2. **Vol surfaces: имплаится 27 базовых активов, в UI — один смайл.** GOLD(242),
   Si(221), SILV(114), RTS(102), BR(84), SBRF/GAZR/CNY/NG/Eu… — богатейший массив
   виден только таблицей-сводкой.
3. **История (16k точек) без единого графика.** КБД по дням, цены 50 акций,
   индексы, RUSFAR — есть presenter, но ChartWidget к ним не подключён.
4. **equity_quotes (786) не имеют explorer** в Market Data (только в портфеле спот).
5. **index_values — мёртвая таблица**, дублирует time_series; удалить или наполнять.

### 1.3 Разрыв инфраструктуры визуализации
`app/chart.py::ChartWidget` (matplotlib) умеет yield curve, vol smile, MC paths,
VaR distribution, survival curve и т.д. — **но не подключён ни к одному presenter'у
на реальных данных.** Весь Market Data / Risk UI сейчас **табличный** (DenseTable).
Это центральный пробел: графический слой существует, но не интегрирован.

## 2. Анализ рынка: что ещё доступно бесплатно

### 2.1 Подтверждено живыми пробами ISS (2026-06-13)
| Данные | Эндпоинт / поле | Зачем | Статус |
|---|---|---|---|
| **Товарные фьючерсы** BR/NG/GOLD/SILV/PLT/SUGAR | forts/securities (ASSETCODE) | commodity-кривые, сезонность; **волы уже имплаятся** | ❌ цены не берём |
| Дивиденды (history) | securities/{secid}/dividends | дивдоходность q для опционов/конвертируемых | 🟡 endpoint есть, не в EOD |
| **Дефолт-флаги** HASDEFAULT/HASTECHNICALDEFAULT | securities/{secid} description | кредитный скрининг, чистка hazard-универсума | ❌ |
| Размер выпуска ISSUESIZE, LISTLEVEL | description | веса/ликвидность, тиринг корп-кривых | 🟡 list_level берём |
| Индексные фьючерсы MIX/RTS/MXI + волы | forts | индексные vol-поверхности (есть волы) | 🟡 волы да, фьючерс-кривая нет |
| Международные SPYF/NASD/ED | forts (волы имплаятся) | прокси глобального риска | 🟡 волы да |
| Open Interest по фьючерсам/опционам | forts marketdata OPENPOSITION | ликвидность, фильтр качества смайла | 🟡 в фильтре, не хранится |

### 2.2 Не на ISS (другие бесплатные / derived)
- **Кредитные рейтинги** АКРА / Эксперт РА — публичные списки (скрейпинг HTML), для
  тиринга корп-кривых лучше LISTLEVEL.
- **ИПЦ / инфляция** Росстат, **ключевые макро** ЦБ (M2, межбанк) — CSV/XML.
- **OFZ-IN база CPI** (индекс облигации) — из ISS bondization linker'ов.
- **Корпоративные действия** (сплиты, оферты) — ISS bondization (оферты уже берём).

## 4. План построения инфраструктуры визуализации

Принцип (как в Stage III): **presenter-слой (чистый, тестируемый) → ChartWidget /
DenseTable (тонкие обёртки)**. Сейчас presenter'ы есть, графический слой — нет.

### Этап V.1 — Графический слой (фундамент, ~3-4 дня)
1. **Chart-ready presenters** в `services/market_views.py`: возвращать массивы
   (x, y, серии) для всех видов — кривые, история, смайлы, поверхности, корреляции.
2. **Подключить ChartWidget к реальным данным** (он уже умеет рисовать):
   - Yield curve overlay: КБД + CORP_T1/2/3 + real на одних осях.
   - Curve history: КБД:5Y и др. как time-series график (+ выбор тенора).
   - Vol smile: сырые точки + SVI-фит поверх (ChartWidget.plot_vol_smile уже есть).
   - Correlation heatmap: матрица факторов (новый метод ChartWidget).
3. **Тесты:** presenter'ы headless (как Stage III); ChartWidget — smoke-конструкция
   под offscreen.

### Этап V.2 — Недостающие explorer'ы (~3-5 дней)
1. **Bond Explorer** (Market Data): YTM×дюрация скаттер (3000 бондов, цвет=тир),
   таблица топ-выпусков, viewer cashflow-расписания (bondization) по выбранному ISIN.
2. **Vol Surface Explorer 2.0**: селектор из 27 базовых активов, 3D-поверхность или
   набор смайлов по экспирациям, ATM-терм-структура, SVI-параметры/rmse.
3. **Equity Explorer**: спот/изменение/объём (786 бумаг), мини-история по выбранной.
4. **Commodity panel**: фьючерсные кривые BR/NG/GOLD (после сбора цен) + волы.

### Этап V.3 — Дашборды и мониторинг (~3-4 дня)
1. **Market Overview dashboard**: КБД-снимок, топ-движения акций, FX, ключевые волы —
   одна сводная картина дня на реальных данных (заменить хардкод demo-дашборда).
2. **Data Health 2.0**: календарь-heatmap покрытия, тренды полноты, ingest-латентность.
3. **Factor Monitor** (Risk): корреляционная heatmap + динамика волатильности факторов.
4. **Time Series Explorer**: любой фактор/кривая-тенор, доходности, rolling-vol.

### Этап V.4 — Расширение сбора под визуализацию (~2-3 дня, параллельно)
1. Дефолт-флаги + ISSUESIZE в instruments (description endpoint) — чистка/тиринг.
2. Товарные фьючерсные кривые (BR/NG/GOLD/SILV) — `ingest_commodity_futures`.
3. Систематические дивиденды (топ-50) → дивдоходность.
4. Веса индексов IMOEX → факторные веса.
5. Удалить/наполнить мёртвую `index_values`.

### Зависимости и порядок
V.1 (графический фундамент) — первым: он разблокирует всё остальное и закрывает
центральный разрыв (16k точек истории и 27 поверхностей без графиков). V.2/V.3
строятся на нём. V.4 идёт параллельно (сбор не блокирует визуализацию имеющегося).

### Принципы (неизменны с фаз 0–III)
- Presenter-first: вся логика подготовки — чистые функции с тестами; Qt тонкий.
- UI→service граница: панели импортируют services/market_views, не curves/* напрямую.
- Реальные данные через app.runtime (live MOEX, fallback demo); никаких хардкодов.
- Любой новый сбор — с quality-проверками и в smoke-тесте ISS-структуры.
